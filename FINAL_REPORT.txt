🚀 DEPLOYMENT STEPS (Quick Reference)

1. PREPARE MONGODB ATLAS
   ├─ Sign up: https://www.mongodb.com/cloud/atlas
   ├─ Create cluster (free M0 tier available)
   ├─ Create database user with password
   ├─ Get connection string: mongodb+srv://...
   └─ Whitelist Render IP: 0.0.0.0/0 (for dev), specific IP for prod

2. SETUP GMAIL (Optional, for password reset emails)
   ├─ Enable 2-Factor Authentication
   ├─ Go to: https://myaccount.google.com/apppasswords
   ├─ Generate app password
   └─ Save 16-character password

3. GENERATE SECRET_KEY
   └─ Run: python -c "import secrets; print(secrets.token_hex(32))"

4. DEPLOY TO RENDER
   ├─ Go to: https://render.com/dashboard
   ├─ Click: New → Web Service
   ├─ Select: Your GitHub repository
   ├─ Set Build Command:
   │  pipe install --upgrade pip && pip install -r requirements.txt
   ├─ Set Start Command:
   │  gunicorn -w 4 -b 0.0.0.0:$PORT wsgi:app
   ├─ Add all environment variables (see section above)
   └─ Click: Deploy

5. VERIFY DEPLOYMENT
   ├─ Wait 3-5 minutes for deployment
   ├─ Visit your app URL
   ├─ Test login
   ├─ Add an expense
   ├─ Check reports page
   └─ Review logs for errors

🆘 COMMON QUESTIONS

Q: What if I don't have a MongoDB Atlas account?
A: Sign up at https://www.mongodb.com/cloud/atlas (free tier available)

Q: Can I use local MongoDB instead?
A: Only for local development. Production requires MongoDB Atlas or similar.

Q: What if I don't want to set up email?
A: Email is optional. Leave EXPENSE_TRACKER_EMAIL* variables empty,
   password reset will fail gracefully.

Q: Do I need to change the SECRET_KEY later?
A: Generate a new one monthly for security, then all users will be logged out.

Q: What if deployment fails?
A: Check Render logs (dashboard → Logs tab) and refer to DEPLOYMENT.md 
   troubleshooting section.

Q: How do I monitor my deployed app?
A: Use Render dashboard → Metrics tab for CPU, memory, network usage.
   Use Logs tab for error messages and debugging.

Q: Can I scale after deployment?
A: Yes. Upgrade Render plan and add more workers in Procfile.
